Modify the existing Qwen experiment pipeline so we can export off-the-shelf pretrained Qwen embeddings for the tracked `dair-ai/emotion` validation split.

## Why this is the correct experiment surface

The existing fine-tuned Qwen embeddings and logits were generated by:

- `experiments/text_model/train_multiclass.py`

That is the experiment entrypoint that currently:

- loads `Qwen/Qwen3-1.7B`
- fine-tunes the classifier head and backbone
- evaluates on a held-out split
- writes `analysis/eval_embeddings.npy`
- writes `analysis/eval_logits.npy`
- writes `analysis/centroids.json`

The downstream bridge/parity layer under:

- `experiments/text/multiclass/dair-ai-emotion/src/run_qwen_bge_parity.py`

only repackages artifacts from an already completed fine-tuned run. It does not perform a fresh pretrained-model forward pass, so it is not the correct place to implement this request.

## User request to satisfy

We now need:

- pretrained, off-the-shelf `Qwen/Qwen3-1.7B`
- no fine-tuning
- run on the repository's tracked validation split
- export 768-dimensional embeddings for that split

This is specifically for the validation set already stored under:

- `experiments/text/multiclass/dair-ai-emotion/data/processed/validation/`

Do not use the historical CSV hold-out reconstruction path for this task. The request is explicitly about the tracked validation split.

## Required implementation

Modify `experiments/text_model/train_multiclass.py` so it supports an extraction-only mode while preserving the current fine-tuning workflow unchanged.

### Add a new execution mode

Add a CLI surface along these lines:

- `--mode train`
- `--mode extract-only`

`train` should remain the default so the existing SLURM jobs keep working.

### Add split selection for processed data

When `--data-root` points at the experiment root `experiments/text/multiclass/dair-ai-emotion`, add a way to choose:

- `train`
- `validation`

Suggested CLI:

- `--data-split train`
- `--data-split validation`

For processed-data mode, resolve:

- `experiments/text/multiclass/dair-ai-emotion/data/processed/<data-split>/texts.jsonl`

Do not force a stratified resplit when `--mode extract-only` is used with `--data-split validation`.

### Extraction-only behavior

In `extract-only` mode:

- load the pretrained Qwen backbone off the shelf from `Qwen/Qwen3-1.7B`
- do not train
- do not create an optimizer
- do not run the epoch loop
- do not save a classifier checkpoint or head weights
- do not write class logits unless they are genuinely meaningful for this mode

The embedding definition must stay aligned with the fine-tuned workflow:

- final transformer hidden state
- last token position
- cast to `float32`
- resulting dimensionality should be 768 for Qwen3-1.7B

### Artifacts to write

Write artifacts under the normal run directory in `experiments/text_model/runs/<run_id>/analysis/`.

At minimum write:

- `validation_embeddings.npy`
- `validation_metrics.json`
- `validation_centroids.json`

`validation_metrics.json` should include:

- dataset size
- embedding dimension
- source split
- model name
- mode = `extract-only`
- `pretrained_only = true`

If labels are available in the validation JSONL rows, compute class centroids from those labels and save them to `validation_centroids.json`.

Also write `run_metadata.json` describing:

- mode
- model reference
- processed split path used
- no fine-tuning was performed

If `train_metrics.json` is not meaningful in extract-only mode, either omit it or write a minimal explicit placeholder. Do not fabricate epoch metrics.

## SLURM contract

The repo already contains a new batch script:

- `hpc/extract_qwen_validation_embeddings.slurm`

Make the implementation support this command shape:

```bash
python experiments/text_model/train_multiclass.py \
    --mode extract-only \
    --data-root experiments/text/multiclass/dair-ai-emotion \
    --data-split validation \
    --run-root experiments/text_model/runs \
    --run-name "${SLURM_JOB_NAME}_${SLURM_JOB_ID}" \
    --batch-size 8 \
    --max-length 256 \
    --model-path Qwen/Qwen3-1.7B
```

If you need to add one or two extra flags to make the implementation cleaner, keep the SLURM change minimal and update the batch file too.
