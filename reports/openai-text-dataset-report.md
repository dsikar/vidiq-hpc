# Dataset Types for Embedding Analyses

We look for **short-text classification corpora** with clean semantic labels (e.g. positive/negative, formal/informal).  The ideal datasets have binary or few categories, balanced examples, and minimal noise or confounds.  In practice this means sentiment or emotion corpora (positive vs negative, happy vs sad), style/formality corpora, and other simple semantic axes like subjective vs objective.  For example, classic sentiment datasets (movie or product reviews), subjectivity sets, and style-transfer corpora are rich in human-interpretable contrasts.  We favor moderate size (thousands to tens of thousands of examples, not millions) so that analysis can focus on semantics rather than data cleaning. Each dataset above is publicly documented and (mostly) accessible via academic release or huggingface.  All have straightforward use terms (academic/public).  They cover our target contrasts: **positive vs negative** (SST, Yelp, Amazon, MR), **formal vs informal** (GYAFC), **subjective vs objective** (Subjectivity), **polite vs rude** (Politeness), **emotional classes** (ISEAR), etc.  In contrast, very large corpora or tasks requiring heavy cleanup (like social media sentiment or multilingual benchmarks) are excluded by design.

## Key Dataset Notes

- **Binary vs Ordinal:** SST-2, MR Polarity, Yelp/Amazon Polarity, and Subjectivity are clean binary-label corpora. ISEAR is multi-way (7 emotions), and Politeness has a quasi-ordinal politeness score (but can be binarized).
- **Continuous Gradients:** Datasets like ISEAR (multiple emotions) and Politeness (score) can be explored for gradient effects, but are less “simple binary.”
- **Potential Confounds:** Many sentiment datasets (Yelp, Amazon) have topic cues (e.g. food words in positive or negative reviews). The subjectivity set mixes movie-review snippets with IMDB plots, so topic style could drive separation. Politeness is specific to request language (so embedding might pick up forum jargon). GYAFC formal/informal may have obvious lexical signals (smileys, slang) that split classes.
- **Size & Accessibility:** SST-2 and the Pang/Lee sets are small (10K or less) and easy to handle. GYAFC is large (110K pairs) but can be subsetted. Yelp/Amazon are huge (hundreds of thousands to millions) – not ideal for initial experimentation unless heavily sampled. All datasets have clear citations and usage notes (see references).

# Top 3 Recommended Datasets

Based on cleanliness, interpretability, and ease of use, the top three picks are:

1. **Stanford Sentiment Treebank (SST-2)**【15†L766-L775】 – A classic binary sentiment dataset (≈11,855 movie-review sentences). It exemplifies a simple “good vs bad” polarity axis. Sentences are short, labels unambiguous, and the dataset is widely studied. Its moderate size and sentence-level nature make it easy to load, embed, and analyze differences between positive/negative. The embedding space likely contains a strong sentiment direction (as many studies suggest), making it ideal for testing whether an LLM’s embeddings capture positive vs negative sentiment.

2. **Pang & Lee Subjectivity Dataset**【42†L75-L77】 – 10,000 sentences evenly split between *subjective* (opinionated) and *objective* (factual) content. This clean binary contrast (subjective vs objective) is intuitively semantic. Since subjective language often co-occurs with sentiment and personal viewpoint, this tests a different axis: does an embedding capture the “opinionated” dimension, distinct from mere topic content? The data is concise (movie review snippets vs plot summaries) and balanced. Its main drawback is that domain differs for the two classes, but for embedding analysis that can itself be interesting: it lets us check if embeddings capture style (subjectivity) even when topics overlap.

3. **GYAFC (Formality)**【4†L26-L29】 – A large corpus of sentence pairs, one informal and one formal (110K total). This directly encodes the “informal vs formal” style axis. Analysis can simply treat each sentence (or take only one side of each pair) with a label. Its size means plenty of data, yet the sentences are fairly short (Yahoo Answers questions/answers). It tests a stylistic contrast rather than sentiment. A risk is that lexical cues (e.g. slang like “lol”) might give away the informal class easily, but that also shows how embedding space might align with style markers. This dataset is ideal for experiments on stylistic directions in embeddings.

These three cover distinct simple semantic dimensions (sentiment, subjectivity, formality). They have clear binary labels and short text lengths. We expect that embeddings of these corpora will reveal clean separations or directions corresponding to those semantics. 

# Suggested Experiments

1. **Linear Probing / Classifier**: For each dataset, extract embeddings of all texts (e.g. the last-layer [CLS] embedding from a transformer or a static embedding average). Train a simple linear (logistic) classifier on a subset of embeddings to predict the class (e.g. positive vs negative). High performance would indicate a linear separation in embedding space. Inspect the learned weight vector (the “direction” separating classes) to see which embedding dimensions or tokens it relies on. *Concrete check:* Train on 80% of SST-2 embeddings, test on 20%. Do the same for GYAFC (formal/informal) and Subjectivity. If accuracy is high, it suggests a clear embedding direction. Examine example sentences near the decision boundary.

2. **Mean-Difference Direction and Projection**: Compute the mean embedding for each class (e.g. μ_pos and μ_neg). The vector Δ = μ_pos – μ_neg is a candidate semantic direction (e.g. sentiment axis). Project all embeddings onto this Δ (i.e. compute cosine or dot-product with Δ). Plot the distributions of projections by class. This tests if a single direction separates classes. *Concrete check:* On SST-2, plot histograms of positive vs negative projections; ideal if they form distinct clusters. Repeat for Subjectivity and Formality. Also project other phrases (not in data) to see if they align with “opinionated” vs “factual” or “formal” vs “informal” as expected.

3. **Dimensionality Reduction Visualization**: Run PCA or t-SNE on embeddings, color points by class label. Simple 2D plots can reveal whether classes form distinct clusters or a gradient. *Concrete check:* Use PCA on embeddings from the subjectivity dataset (5000 of each class). Plot with color=subjective/objective. A separation or linearly separable pattern would support a clean semantic axis. Repeat for SST-2 (positive/negative) and maybe GYAFC (formal/informal). Ensure to use the same pre-trained model embeddings for all.

4. **Lexical Correlation / Lexicon Testing**: Identify whether known indicator words align with the embedding directions. For sentiment datasets (SST-2), check if common positive adjectives (“great”, “excellent”) have embeddings whose projection on Δ is strongly positive, and negative words (“bad”, “terrible”) project negatively. For subjectivity, check if first-person pronouns or opinion verbs lean one way. *Concrete check:* Take a sentiment lexicon (e.g. “good, amazing, excellent” vs “bad, horrible”) and compute each word’s embedding and its projection on the learned sentiment axis Δ. Verify that they match expected polarities. This ties embedding directions to human-interpretable semantics.

5. **Embedding Arithmetic / Analogy**: Use simple “embedding arithmetic” tests. For example, for formal⇔informal, see if embedding(“Please explain X”, formal) + (embedding difference between formal and informal neutral phrase) ≈ embedding(“Explain X”, informal) (conceptual analogies). More systematically, see if adding the Δ_formality vector to an informal sentence embedding pushes it toward the formal cluster (and vice versa). *Concrete check:* Pick a few paired sentences from GYAFC (or similar style pairs) and verify if their embeddings differ roughly by a constant vector. For sentiment, try “embedding(‘This movie is great’) – embedding(‘movie’) + embedding(‘awful’)” and see if it approximates a strong negative. These tests probe linearity of embedding semantics.

Each experiment should be done across the shortlisted datasets to compare how “cleanly” different semantics emerge in embedding space. For example, if a linear classifier easily separates positive vs negative on SST-2 but fails on ISEAR’s subtle emotions, that suggests sentiment is a more robust direction. By examining weights, projections, and clusters, we build intuition on how an LLM encodes these simple semantic axes.
